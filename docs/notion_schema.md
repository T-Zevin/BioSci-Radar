# Suggested Notion Schema

Create a Notion database called `Papers` and share it with your integration.

Recommended fields:

| Property | Type | Purpose |
|---|---|---|
| Title | Title | Paper title |
| Status | Select | New, To Read, Reading, Read, Useful, Ignore |
| Priority | Select | High, Medium, Low |
| Score | Number | Overall recommendation score |
| Paper Type | Select | Bio Study, Bioinformatics Method, ML Algorithm, Review, Dataset |
| Source | Select | PubMed, bioRxiv, medRxiv, arXiv |
| Year | Number | Publication year |
| Published | Date | Publication date |
| URL | URL | Source link |
| DOI | Rich text | DOI |
| PMID | Rich text | PubMed ID |
| Omics Type | Multi-select | scRNA, scATAC, spatial, proteomics, metabolomics, microbiome, etc. |
| Disease | Multi-select | LUAD, cancer, tumor, immune, etc. |
| ML Area | Multi-select | GNN, Transformer, VAE, SSL, Contrastive, Survival, Causal, Foundation Model |
| Topics | Multi-select | Matched configured topics |
| Code Available | Checkbox | Whether code appears to be available |
| Data Available | Checkbox | Whether public data appears to be available |
| Transfer Potential | Select | High, Medium, Low |
| Reproducibility | Select | High, Medium, Low, Unknown |
| Why Relevant | Rich text | Short reason generated by the scorer |
| Abstract | Rich text | Source abstract |

The sync command creates pages with these properties when they exist. If a property is missing, Notion rejects that page update, so keep the property names exact for the first MVP.
